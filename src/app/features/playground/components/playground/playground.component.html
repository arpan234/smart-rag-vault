Query Tokenization & Vector Embedding

Turns the raw text query into a mathematical coordinate array.

How it works
Embeddings are float arrays representing the semantic meaning of text. Words with similar meanings are plotted close to each other in a high-dimensional vector space. In Local VSM mode, the dimensions correspond to exact word coordinates (TF-IDF weights). In API mode, they represent compressed latent concepts (768 or 1536 dimensions).

Vector Database Similarity Scan

Calculates similarity angles between query vector and all stored document chunk vectors.

How it works
The database does a linear scan, calculating the **Cosine Similarity** (from 0% to 100%) between the query vector q and each document chunk vector d. Chunks are ranked by score. Only those exceeding your minimum similarity threshold (e.g. 30%) are retrieved.

Prompt Decoration & Augmentation

Glues the system instructions, matched document context chunks, and user query together.

How it works
An LLM has no memory of your uploaded files. To answer your query, we must **augment** the prompt. We wrap the raw matching text chunks into a template containing system guardrails ("Use the context above to answer..."), formatting it as a single decorated prompt string.

LLM Answer Synthesis & Generation

The model processes the structured prompt containing context facts and synthesizes the response.

How it works
The LLM reads the system instructions, the retrieved facts, and your question. Because the facts are inside its context window, it can answer with high precision ("grounded answer") and avoid generating false information ("hallucinations").
